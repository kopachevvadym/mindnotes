# mindnotes — архітектура (станом на 2026-07-02)

Нотатник потоку думок під час читання. Два режими: **Потік** (миттєве захоплення
сирих думок у сесії) і **Синтез** (пізніша організація думок у групи, що зріють
до ідей). AI ще не підключений; організація повністю ручна.

## Стек і оркестрація

| Шар | Технології |
|---|---|
| Монорепо | pnpm workspaces + turborepo (`dev`, `build`, `typecheck`, `test`) |
| Бекенд | Hono на **bun**, drizzle-orm, `bun:sqlite` (локальний файл, WAL, FK pragma) |
| Спільний код | `@mindnotes/schema`: drizzle-таблиці + zod DTO-схеми + константи |
| Фронтенд | Vite + React 19 + TS, tailwind v4, radix/shadcn-подібні ui, tanstack router (file-based) + query, zustand (стан палітри пошуку), оптимістичні мутації |

```
apps/api        Hono, REST, drizzle; src: index (усі роути), db, env, migrate, seed, serialize
apps/web        Vite React SPA; src: routes → pages → components, lib (api-client, queries, mutations, format), store
packages/schema tables.ts (drizzle), api.ts (zod DTO), constants.ts
```

## Модель даних (SQLite, drizzle)

```
session ──< thought >──< context_thought >── context
```

- **session** — `id` (uuid-рядок), `title` (NULL дозволено), `startedAt`, `createdAt`
  (unix ms → Date у drizzle). Сесію можна не називати.
- **thought** — `sessionId` (FK, cascade), `body`, `archived` (bool), `createdAt`.
  Архівування ховає з основного потоку, але не видаляє.
- **context** — група на градієнті зрілості: `thesis` NULL ⇒ «контекст» (тематична
  група), заповнена ⇒ «ідея». Один обʼєкт, не окремі сутності. Народжується «знизу» —
  з думки-насінини.
- **context_thought** — M2M (складений PK). Думка може живити кілька груп;
  крос-сесійний синтез — центральний сценарій.

Міграції: `0000` (початкова схема, ще з таблицею `idea`), `0001` (перейменування
`idea`→`context`, `idea_thought`→`context_thought` через `ALTER TABLE RENAME`,
дані збережено).

Правила життєвого циклу:
- Редагування/видалення думки — лише в межах **вікна 15 хв** від створення
  (`THOUGHT_EDIT_WINDOW_MIN`, спільна константа: бекенд enforce'ить, фронт ховає дії).
- Видалення групи не чіпає думки (транзакція: лінки → група). Видалення сесії
  каскадом прибирає її думки.

### Ще не реалізоване з ухваленої моделі (CLAUDE.md, 2026-06-29)

`thought.parentId` (дерево «думка від думки» в межах сесії) — ухвалено, але в
схемі й UI поки відсутнє.

## API (REST, JSON)

Конвенції: вхід валідується zod-схемою (`safeParse` → 400 `invalid_body`); вихід
**теж** проганяється через спільну DTO-схему перед `c.json` (fail-fast на дрейф
контракту). Дати серіалізуються в ISO-рядки (`serialize.ts`). Помилки — завжди
JSON `{ error: "<code>" }`: і явні (`session_not_found`, `edit_window_closed`…),
і `onError`/`notFound` (`internal`, `not_found`). CORS лише для `WEB_ORIGIN`.
Багатокрокові записи — синхронні транзакції `bun:sqlite`.

| Метод і шлях | Дія |
|---|---|
| GET `/health` | ok |
| GET `/search?q=` | пошук: сесії за назвою, групи за тезою/превʼю, думки за текстом; регістронезалежно для кирилиці (фільтр у JS — SQLite `lower()` згортає лише ASCII); ліміти 5/5/8 |
| GET `/sessions` | список + `thoughtCount` (включно з архівними), новіші зверху |
| POST `/sessions` | створити порожню (без назви) |
| GET `/sessions/:id` | `{ session, thoughts[] }` asc; кожна думка з `contextId` найновішої групи (мітка-двері) або null |
| PATCH `/sessions/:id` | перейменувати (`title` може бути null) |
| DELETE `/sessions/:id` | видалити (cascade думки) |
| POST `/sessions/:id/thoughts` | створити думку |
| PATCH `/thoughts/:id` | АБО `{archived}` (будь-коли), АБО `{body}` (лише у вікні; інакше 403) |
| DELETE `/thoughts/:id` | лише у вікні (інакше 403) |
| POST `/contexts` | народити групу з `seedThoughtId` (thesis=null), повертає групу+думки |
| GET `/contexts` | список: thesis, `thoughtCount`, `previewBody` (найраніша думка групи), новіші зверху |
| GET `/contexts/:id` | група + думки (кожна з `sessionTitle` — джерелом) |
| PATCH `/contexts/:id` | оновити тезу (trim; порожня → null) |
| POST `/contexts/:id/thoughts` | втягнути наявну думку (ідемпотентно, `onConflictDoNothing`) |
| DELETE `/contexts/:id/thoughts/:thoughtId` | відчепити думку (сама лишається) |
| DELETE `/contexts/:id` | видалити групу, думки лишаються |

Env (`apps/api/src/env.ts`, zod-валідація, єдиний `.env` у корені): `DATABASE_URL`
(шлях до sqlite-файлу відносно apps/api), `PORT` (3000), `WEB_ORIGIN` (5173).

## Фронтенд

**Потік даних:** file-based роути (`routes/` → `routeTree.gen.ts`) → сторінки
(`pages/`) → `useQuery(queries.ts)` → єдиний `api-client.ts` (базовий URL лише з
`VITE_API_BASE_URL`, кожна відповідь — `schema.parse`). Мутації (`mutations.ts`) —
усі з оптимістичним оновленням кешу: onMutate знімок → підміна, onError відкат,
onSettled інвалідація. Клієнт читає `{error: code}` з тіла невдалої відповіді й
мапить на людські тексти українською (`ApiError.code` + `ERROR_MESSAGES`).

**Маршрути:** `/` (список сесій), `/sessions/:id` (Потік),
`/sessions/:id/triage` (розбір; un-nested через `$sessionId_.triage.tsx`),
`/contexts` (список груп), `/contexts/:id` (сторінка групи/ідеї). Спільний каркас
`__root.tsx`: колонка `max-w-reading`, навігація «Сесії · Ідеї», палітра пошуку.

**Потік** (`SessionPage` → `ThoughtStream`, `CaptureBar`):
- Захоплення: Enter — сабміт, Shift+Enter — новий рядок; поле очищається одразу.
  Мережеві/5xx збої тихо ретраяться двічі; якщо не вийшло — текст повертається в
  поле І під баром зʼявляється видимий рядок «Не збереглося…».
- Автоскрол до нової думки; маркери «тиша N хв» при паузі ≥15 хв; відносний час
  (на hover; на дотикових пристроях видимий завжди — варіант `hoverless`).
- Дії над думкою — тихе меню «⋮»: нова ідея, у наявну (пікер-шторка з превʼю і
  пошуком), редагувати/видалити (лише у вікні), архівувати. Мітка-двері → група.
- Лінк «розібрати N думок →», поки в сесії є активні думки поза групами.
- Порожня покинута сесія (без назви і думок) авто-видаляється при переході до списку.

**Розбір** (`TriagePage`): нерозібрані думки проходяться по одній, як inbox.
Черга — знімок на вході. Дії: нещодавні групи одним тапом (MRU, клавіші 1–5),
нова ідея (N), повний пікер, архів (A), пропустити (пробіл), Esc — назад;
прогрес «N із M». Гарячі клавіші ігнорують ввід у полях і відкриті діалоги.

**Синтез** (`ContextPage`): теза — інлайн-редагований лід («Сформулюй ідею…»);
думки з посиланням на сесію-джерело; «відчепити»; видалення з підтвердженням.
Список груп (`ContextsListPage`) — двома секціями «Ідеї» / «Контексти»; групи без
тези показуються превʼю найранішої думки.

**Пошук** (`SearchPalette` + `store/search-store.ts`): ⌘K/Ctrl+K
(layout-незалежно, `e.code`) або кнопка в шапці сесії; дебаунс 200 мс, групи
результатів, ↑↓/Enter, «знайти й перейти».

## Механізми якості

- Один контракт на обох боках: zod-схеми з `@mindnotes/schema` валідують відповідь
  і на бекенді (перед віддачею), і на фронті (в api-client).
- `pnpm test` (turbo → bun test): інтеграційні тести API на in-memory SQLite
  (усі роути, вікно 15 хв, ідемпотентність, каскади, пошук) + юніт-тести
  `format.ts`. Тести web мають окремий `tsconfig.test.json`, щоб bun-глобали не
  текли в DOM-код.
- `pnpm typecheck` через turbo по всіх пакетах.
- Міграції drizzle (`db:generate` / `db:migrate`); `db:seed` — ідемпотентний сид
  «Викрадена увага» (фіксований `SEED_SESSION_ID`).

## Відомі хвости

- `session.startedAt` і `session.createdAt` фактично дублюються (обидва — момент
  створення; окремого «старту» ніхто не ставить). Кандидат на злиття при
  наступній міграції.
- Лічильник у списку сесій рахує думки разом з архівними — більше, ніж видно в
  потоці.
- Лінтера/форматера немає (пропозиція B3 — Biome — не ухвалена).
