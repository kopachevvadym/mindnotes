# mindnotes — архітектура (станом на 2026-07-02)

Нотатник потоку думок під час читання. Два режими: **Потік** (миттєве захоплення
сирих думок у сесії) і **Синтез** (пізніша організація думок в ідеї). AI ще не
підключений; організація повністю ручна.

## Стек і оркестрація

| Шар | Технології |
|---|---|
| Монорепо | pnpm workspaces + turborepo (`dev`, `build`, `typecheck`) |
| Бекенд | Hono на **bun**, drizzle-orm, `bun:sqlite` (локальний файл, WAL, FK pragma) |
| Спільний код | `@mindnotes/schema`: drizzle-таблиці + zod DTO-схеми + константи |
| Фронтенд | Vite + React 19 + TS, tailwind v4, radix/shadcn-подібні ui, tanstack router (file-based) + query, оптимістичні мутації |

```
apps/api        Hono, REST, drizzle; src: index (усі роути), db, env, migrate, seed, serialize
apps/web        Vite React SPA; src: routes → pages → components, lib (api-client, queries, mutations, format)
packages/schema tables.ts (drizzle), api.ts (zod DTO), constants.ts
```

## Модель даних (SQLite, drizzle)

```
session ──< thought >──< idea_thought >── idea
```

- **session** — `id` (uuid-рядок), `title` (NULL дозволено), `startedAt`, `createdAt`
  (unix ms → Date у drizzle). Сесію можна не називати.
- **thought** — `sessionId` (FK, cascade), `body`, `archived` (bool), `createdAt`.
  Архівування ховає з основного потоку, але не видаляє.
- **idea** — `thesis` (NULL, доки не сформульована), `createdAt`. Народжується
  «знизу» — з думки-насінини.
- **idea_thought** — M2M (складений PK). Думка може живити кілька ідей;
  крос-сесійний синтез — центральний сценарій.

Правила життєвого циклу:
- Редагування/видалення думки — лише в межах **вікна 15 хв** від створення
  (`THOUGHT_EDIT_WINDOW_MIN`, спільна константа: бекенд enforce'ить, фронт ховає дії).
- Видалення ідеї не чіпає думки (транзакція: лінки → ідея). Видалення сесії
  каскадом прибирає її думки.

### Розбіжність з ухваленою моделлю (CLAUDE.md, 2026-06-29)

Ухвалений кістяк описує таблицю **`context`** (порожня теза ⇒ «контекст», заповнена ⇒
«ідея» — той самий обʼєкт на градієнті зрілості) та **`thought.parentId`** (дерево
«думка від думки» в межах сесії). У коді натомість таблиця називається `idea`, а
`parentId` не існує. Тобто документація-модель випереджає код: перейменування і
дерево думок ще не реалізовані.

## API (REST, JSON)

Конвенції: вхід валідується zod-схемою (`safeParse` → 400 `invalid_body`); вихід
**теж** проганяється через спільну DTO-схему перед `c.json` (fail-fast на дрейф
контракту). Дати серіалізуються в ISO-рядки (`serialize.ts`). Помилки — `{ error:
"<code>" }` (`session_not_found`, `edit_window_closed`…). CORS лише для
`WEB_ORIGIN`. Багатокрокові записи — синхронні транзакції `bun:sqlite`.

| Метод і шлях | Дія |
|---|---|
| GET `/health` | ok |
| GET `/sessions` | список + `thoughtCount` (включно з архівними), новіші зверху |
| POST `/sessions` | створити порожню (без назви) |
| GET `/sessions/:id` | `{ session, thoughts[] }` asc; кожна думка з `ideaId` найновішої ідеї (мітка-двері) або null |
| PATCH `/sessions/:id` | перейменувати (`title` може бути null) |
| DELETE `/sessions/:id` | видалити (cascade думки) |
| POST `/sessions/:id/thoughts` | створити думку |
| PATCH `/thoughts/:id` | АБО `{archived}` (будь-коли), АБО `{body}` (лише у вікні; інакше 403) |
| DELETE `/thoughts/:id` | лише у вікні (інакше 403) |
| POST `/ideas` | народити ідею з `seedThoughtId` (thesis=null), повертає ідею+думки |
| GET `/ideas` | список: thesis, `thoughtCount`, новіші зверху |
| GET `/ideas/:id` | ідея + думки (кожна з `sessionTitle` — джерелом) |
| PATCH `/ideas/:id` | оновити тезу (trim; порожня → null) |
| POST `/ideas/:id/thoughts` | втягнути наявну думку (ідемпотентно, `onConflictDoNothing`) |
| DELETE `/ideas/:id/thoughts/:thoughtId` | відчепити думку (сама лишається) |
| DELETE `/ideas/:id` | видалити ідею, думки лишаються |

Env (`apps/api/src/env.ts`, zod-валідація, єдиний `.env` у корені): `DATABASE_URL`
(шлях до sqlite-файлу відносно apps/api), `PORT` (3000), `WEB_ORIGIN` (5173).

## Фронтенд

**Потік даних:** file-based роути (`routes/` → `routeTree.gen.ts`) → сторінки
(`pages/`) → `useQuery(queries.ts)` → єдиний `api-client.ts` (базовий URL лише з
`VITE_API_BASE_URL`, кожна відповідь — `schema.parse`). Мутації (`mutations.ts`) —
усі з оптимістичним оновленням кешу: onMutate знімок → підміна, onError відкат,
onSettled інвалідація (реконсайл із сервером).

**Маршрути:** `/` (список сесій), `/sessions/:id` (Потік), `/ideas` (список),
`/ideas/:id` (Синтез: теза-лід + думки з джерелами). Спільний каркас `__root.tsx`:
колонка `max-w-reading`, навігація «Сесії · Ідеї».

**Ключові UX-механіки Потоку** (`SessionPage` → `ThoughtStream`, `CaptureBar`):
- Захоплення: Enter — сабміт, Shift+Enter — новий рядок; поле очищається одразу,
  на помилку текст тихо повертається. Фокус у полі приглушує потік (dimmed).
- Автоскрол до нової думки; маркери «тиша N хв» при паузі ≥15 хв; відносний час,
  що проявляється при hover.
- Дії над думкою — тихе меню «⋮»: нова ідея з думки, у наявну ідею (пікер-шторка
  з пошуком за тезою), редагувати/видалити (лише у вікні 15 хв), архівувати.
- Мітка-двері під думкою → сторінка ідеї, яку вона живить.
- Порожня покинута сесія (без назви і думок) авто-видаляється при переході до списку.

**Синтез** (`IdeaPage`): теза — інлайн-редагований лід («Сформулюй ідею…»);
список думок з посиланням на сесію-джерело; «відчепити» думку; видалення ідеї з
підтвердженням (шторка).

## Механізми якості

- Один контракт на обох боках: zod-схеми з `@mindnotes/schema` валідують відповідь
  і на бекенді (перед віддачею), і на фронті (в api-client).
- `pnpm typecheck` через turbo по всіх пакетах. Тестів і лінтера **немає**.
- Міграції drizzle (`drizzle/`, наразі одна `0000`); `db:generate` / `db:migrate` /
  `db:seed` (ідемпотентний seed «Викрадена увага», фіксований `SEED_SESSION_ID`).

## Відомі хвости в коді

- `apps/web/src/store/capture-store.ts` — мертвий (zustand-стор ніде не імпортується;
  CaptureBar живе на локальному useState).
- `apps/web/src/components/ui/input.tsx` — мертвий (не імпортується).
- Кнопка пошуку «⌘K» у `SessionHeader` — заглушка без обробника.
- Коментар до `SEED_SESSION_ID` («корінь редіректить на неї») застарів — корінь
  тепер рендерить список сесій.
- `session.startedAt` і `session.createdAt` фактично дублюються (обидва — момент
  створення; окремого «старту» ніхто не ставить).
